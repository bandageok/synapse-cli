# Task: Improve test coverage for C.C.Claw v0.2.0

## Goal
Write tests for core modules and tools to push coverage from ~30% to >50%.

## Files to test (priority order):

### Priority 1 (core, highest impact):
1. `src/tools/FileReadTool.ts` - read file content
2. `src/tools/FileWriteTool.ts` - write file content  
3. `src/tools/BashTool.ts` - execute bash commands
4. `src/tools/GlobTool.ts` - file pattern search
5. `src/tools/GrepTool.ts` - content search

### Priority 2 (core modules):
6. `src/tools/PowerShellTool.ts` - PS commands
7. `src/tools/TodoWriteTool.ts` - task management
8. `src/soul/Heartbeat.ts` - task scheduler
9. `src/soul/SoulLoader.ts` - SOUL.md loader
10. `src/tools/TaskTool.ts` - sub-agent spawning

### Test guidelines:
- Use vitest: `import { describe, it, expect, beforeEach, afterEach, vi } from 'vitest'`
- Tests go in `tests/<name>.test.ts`
- Use tmpdir for file operations: `import { tmpdir } from 'os'`, `import { join } from 'path'`
- Mock fetch for network tools: `vi.stubGlobal('fetch', vi.fn())`
- Mock `fs` where needed
- Each test file should have 3-8 test cases
- Test both success and failure paths

## Run verification:
```
npx vitest run tests/<new_test>.test.ts
```

```typescript
});
```
